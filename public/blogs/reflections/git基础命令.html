git基础命令

kellen 2020-07-19 git

# 配置用户名和邮箱

--global为全局参数，表明所有本地git仓库都会使用这个配置

git config --global user.name "yourname"

git config --global user.email "your_email@youremail.com"
1
2
3

# 生成密钥（SSH Key）

ssh-keygen -t rsa -C "your_email@youremail.com"
1

# 验证是否连接成功

ssh -T git@github.com
1

# git init

初始化git

git init
1

# git add

提交所有变化

git add -A
1

提交被修改(modified)和被删除(deleted)的文件，不包括新文件(new)

git add -u
1

提交新文件(new)和被修改(modified)的文件，不包括被删除(deleted)的文件

git add .
1

# git commit

-m表示后面直接输入提交代码的信息

git commit -m "message"
1

其他功能如-m参数，加的-a参数可以将所有已跟踪文件中的执行修改或删除操作的文件都提交到本地仓库， 即使它们没有经过git add添加到暂存区,注意，新加的文件（即没有被git系统管理的文件） 是不能被提交到本地仓库的。建议一般不要使用-a参数，正常的提交还是使用git add先将要改动的文件添加到暂存区， 再用git commit 提交到本地版本库。

git commit -a -m "message"
1

# 连接远程仓库

git remote add origin "git@github.com:yourname/repositoryname.git"
1

# 断开远程仓库

git remote remove origin
1

# 上传代码

-u为可选参数，使用-u以后就直接git push就行，origin为连接远程仓库的别名，master为默认分支

git push -u origin master
1

# 下拉代码

下拉分支为master的代码

git pull origin master
1

# 查看git状态

git status
1

# 查看git日志

git log
1
